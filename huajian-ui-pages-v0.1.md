# 化简后台页面规划 v0.1

## 1. 设计参考

用户后台和平台总后台可以参考 Art Design Pro 这类现代后台模板的设计方向：

- 左侧主导航。
- 顶部工具栏。
- 内容区卡片化。
- 表格、筛选、弹窗、抽屉、步骤条清晰。
- 色彩现代，视觉轻盈。
- 支持暗色模式。
- 支持多标签页工作区。
- 操作按钮明确。

Art Design Pro 项目地址：

```text
https://github.com/Daymychen/art-design-pro
```

当前判断：

```text
适合作为化简后台前端基座。
```

原因：

- MIT 开源协议，允许商用和二次开发，但需要保留版权声明。
- 技术栈是 Vue3、TypeScript、Vite、Element Plus、Tailwind CSS。
- 内置数据展示、表单、主题切换、页面过渡、图表等后台常用能力。
- 有精简脚本，可以去掉演示数据后作为业务项目起点。
- 视觉比传统后台更现代，适合化简这种 AI 建站 SaaS 产品。

需要注意：

- 它只能作为后台管理端，不作为生成网站的前台模板。
- 化简前台模板仍然坚持 HTML/CSS/JavaScript 静态资源规范。
- 不要把 Art Design Pro 的复杂演示页面全部保留进产品。
- 要清理 demo、重写路由、菜单、权限和接口层。

但化简不是展示型官网，而是长期使用的 SaaS 工作台，所以要避免过度装饰。

设计关键词：

```text
现代
清爽
高信息密度
低学习成本
AI 友好
发布流程清楚
适合站群管理
```

## 1.1 Art Design Pro 改造方案

### 1.1.1 使用范围

Art Design Pro 用于：

```text
平台总后台
客户管理后台
AI 助手工作台
站点发布管理
模板中心
内容和商品管理
```

不用于：

```text
客户生成出来的网站前台
静态模板系统
商品详情页前台
文章详情页前台
SEO 落地页
```

### 1.1.2 前后端分工

```text
Art Design Pro
  负责后台界面、表格、表单、路由、状态管理、请求封装。

PHP API
  负责客户、站点、文章、商品、AI、发布、部署等业务接口。

静态生成器
  负责把内容和模板生成 HTML 文件。
```

推荐结构：

```text
huajian/
  admin-web/       Art Design Pro 改造后的后台前端
  server/          PHP 后端 API
  worker/          静态生成和 AI 任务
  templates/       化简前台模板
  sites/           生成后的站点文件
```

### 1.1.3 需要保留的能力

```text
登录页
主布局
左侧菜单
顶部栏
标签页
暗色模式
主题配置
表格封装
表单封装
图表组件
路由权限
Axios 请求封装
错误页
```

### 1.1.4 需要清理的内容

```text
演示业务页面
无关图表页面
无关组件示例
Mock 数据
默认菜单
默认权限数据
默认品牌信息
```

清理后重建：

```text
平台后台菜单
客户后台菜单
化简品牌视觉
化简 API 接口
化简权限模型
化简状态标签
```

### 1.1.5 技术选型结论

后台前端：

```text
Vue3 + TypeScript + Vite + Element Plus + Tailwind CSS
```

后端接口：

```text
PHP + MySQL
```

前台生成站点：

```text
HTML + CSS + JavaScript
```

这套组合不会冲突：

- Vue 后台提升管理效率。
- PHP 后端符合你的熟悉度。
- HTML 静态前台保证 SEO 和站群性能。

### 1.1.6 后台命名建议

项目内部可以命名：

```text
admin-web      后台前端
admin-api      后台接口
site-builder   静态生成器
site-public    生成站点
```

页面标题：

```text
化简平台总后台
化简站点后台
```

## 3. 视觉风格建议

### 3.1 色彩

主色建议：

```text
主色：#1677ff
辅助青色：#13c2c2
成功：#52c41a
警告：#faad14
危险：#ff4d4f
背景：#f5f7fb
卡片：#ffffff
正文：#1f2937
次级文字：#6b7280
边框：#e5e7eb
```

说明：

- 蓝色适合 SaaS、科技、后台系统。
- 青色可用于发布、同步、AI、生成等动作。
- 不建议大面积紫色渐变，容易像营销页。

### 3.2 字体和间距

```text
字体：系统默认字体
正文字号：14px
表格字号：13px 或 14px
页面标题：20px 或 22px
卡片圆角：8px
按钮圆角：6px
页面间距：16px / 24px
表格行高：48px
```

### 3.3 组件风格

推荐：

- 卡片承载统计数据。
- 表格承载管理列表。
- 抽屉承载详情。
- 弹窗承载短表单。
- 步骤条承载发布流程。
- 标签承载状态。
- 分段按钮承载视图切换。
- 图标按钮承载刷新、复制、预览、删除。

避免：

- 页面里套太多卡片。
- 大量渐变背景。
- 过大的空白 Hero。
- 复杂拖拽优先于稳定表单。
- 用营销页风格做后台。
